publishing releases via the admin page · full guide: TEAM-GUIDE.md
WHERE: https://<your-site>/admin.html LOGIN: sontrarecords@gmail.com · password — ask the label owner ⚠ shared login: tell the team chat before editing

1Prepare 2 files first

  • 🎨 Artwork — JPG/PNG, square, ≥ 1400×1400
  • 🎵 Audio — under 50 MB (hard limit). A full WAV won't fit — convert it:

afconvert -f m4af -d aac -b 256000 "in.wav" "out.m4a"

(Terminal, any Mac.) Keep the WAV master in the archive — upload the m4a.

2Sign in

  • Open the admin page → enter email + password → Sign in
  • Left: “New release” form
  • Right: Catalog — everything already on the site

3Fill the form

  • Catalog — release number, e.g. STR002 (names the files)
  • Title / Artist / Year / Genre — as on the release
  • BPM — track tempo (powers the beat grid)
  • Beat offset — seconds to the first downbeat (0 if it starts on the beat)
  • Artwork + Audio — pick the 2 prepared files

4Save → it's live

  • Click Save → “Uploading artwork… audio…” → Saved ✓
  • The release is immediately on the public site — no deploy, no git
  • Verify: open the site → find it → press play

5Edit a release

  • Catalog list → Edit → change fields → Save
  • Leave file pickers empty to keep the current audio/artwork
  • Pick a new file only to replace it

6Delete a release

  • Catalog list → Delete → confirm
  • Removes it from the site immediately
  • No undo — when unsure, ask the owner first
Trouble signing in / 50 MB limit / anything weird → ping the label owner. · Admin page is internal — don't share the URL.
